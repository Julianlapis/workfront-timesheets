# workfront-timesheets

A Claude Code plugin that fills your Adobe Workfront timesheets semi-autonomously — so you do almost nothing and don't get the errors that plague manual entry.

It gathers time evidence from your **calendar, Slack, and Claude session history**, proposes plausible weekly hour splits, fills the Workfront grid via the [`dev-browser`](https://www.npmjs.com/package/dev-browser) CLI with real-keystroke persistence, **verifies every cell actually saved** (Workfront silently drops entries in several ways), logs anything it can't resolve, and **submits only after you review a proof pack.**

## Why it exists

Workfront's timesheet grid has a set of non-obvious failure modes that cost hours of manual fighting:

- Synthetic value injection updates the visible total but **doesn't persist** — only real keystrokes commit.
- **General Time rows silently revert** unless the week also has a real project entry.
- Autosave is debounced — navigate away too fast and the save **aborts**.
- The grid **virtualizes** rows and **reuses row labels** across project sections.
- Some rows are **disabled** even when the project is on your sheet (you're on the project but not a loggable task).

This plugin encodes all of those as hard-won gotchas and defends against them mechanically (reload-verify, idempotency guards, disabled-row detection).

## Prerequisites

This plugin **drives a real browser** — it does not ship one. You need:

- **[`dev-browser`](https://www.npmjs.com/package/dev-browser)** — the CLI that runs the fill. **Required, not bundled** (a plugin shouldn't vendor a third-party CLI). Install: `npm install -g dev-browser && dev-browser install`
- **Google Chrome**, launched with remote debugging and logged into Workfront: `open -a "Google Chrome" --args --remote-debugging-port=9222 --restore-last-session` (the skill walks you through this in Phase 3).
- **Claude Code MCPs** for **Slack** and **Google Calendar** (the read-only evidence layer). Without them you can still fill from a manual allocation; the evidence layer is just degraded.

The skill runs a STEP 0 preflight that checks for all of these and stops with a clear message if anything's missing — so you won't fail cryptically mid-fill.

## Install

1. Install the prerequisites above.
2. Install the plugin (via your Claude Code plugin marketplace, or point Claude Code at this repo).
3. Copy `config.example.yaml` → `config.yaml` and fill it in. **This is the only file you edit.**
4. Run `/workfront` and say what you want ("fill last month", "do this week").

## What's yours vs. what's the tool

| Layer | Where | You edit? |
|---|---|---|
| **Mechanism** — iframe handling, keystroke fill, persistence rules, virtualization, verify | `skills/workfront/` | No |
| **Policy** — hour targets, the no-fold rule, submit-after-review gate | `config.yaml` | Set once |
| **Per-run data** — which weeks, the allocation PLAN | generated by the agent in Phase 2 | No |

## Hard guarantees

- **Never folds one project's hours onto another project's line.** Unloggable hours get *parked*, not misattributed.
- **Never auto-submits.** You approve a proof pack (allocation table + verify log + screenshot + blocker list) first.
- **Never reports a week done unless a reload confirms every cell persisted.**

## Files

- `skills/workfront/SKILL.md` — the procedure + the gotcha catalog.
- `skills/workfront/fill-week.mjs` — the real-keystroke filler (config block + PLAN; in-code idempotency + reload-verify).
- `skills/workfront/scripts/condense_cal.py` — Google Calendar JSON → compact per-event lines.
- `skills/workfront/scripts/claude_activity.py` — per-week Claude-session activity histogram per project.
- `config.example.yaml` — copy to `config.yaml`.
